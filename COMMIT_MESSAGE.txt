BEHAVIOR:

First-time technician login:
  1. Lands on dashboard
  2. SelectTechnicianModal displays unclaimed profiles
  3. Selects their profile → updates technicians.user_id
  4. Redirects to /technicians/[id]
  5. Views own profile with editable status

Returning technician login:
  1. Auth fetches linked technician
  2. Auto-redirects to /technicians/[id]
  3. Lands directly on profile page

Technician profile viewing:
  - Can edit own status via inline form
  - Cannot add jobs (button hidden)
  - Can view assigned jobs and history
  - Cannot access other technician profiles

DATABASE:

Assumes technicians table has:
  - user_id (UUID, nullable)
  - org_id (UUID, required)
  - status (text enum: 'available', 'busy', 'on-leave')

Queries used:
  - SELECT * FROM technicians WHERE org_id=? AND user_id IS NULL
  - UPDATE technicians SET user_id=? WHERE id=? AND org_id=?
  - SELECT id FROM technicians WHERE user_id=? AND org_id=?
  - UPDATE technicians SET status=? WHERE id=? AND org_id=?

RLS POLICIES (recommended for production):

-- Allow technicians to read their own row
CREATE POLICY technicians_own_access ON technicians
FOR SELECT USING (user_id = auth.uid() OR user_id IS NULL);

-- Prevent technicians from modifying user_id (admin-only)
CREATE POLICY technicians_cannot_modify_user_id ON technicians
FOR UPDATE USING (user_id = auth.uid())
WITH CHECK (user_id = auth.uid());
