### 🗄️ Part 3: Database Schema

#### New Columns Added to `quotations` Table:
- `quote_no` - Quotation number (QT-001, QT-002, etc.)
- `client_district` - Customer district
- `client_state` - Customer state
- `client_pin_code` - Customer pin code
- `subject` - Quotation subject
- `body_text` - Letter body text
- `sgst` - SGST amount (9% of subtotal)
- `cgst` - CGST amount (9% of subtotal)
- `grand_total` - Total with taxes
- `valid_till` - Quotation validity date

#### Removed Columns:
- `customer_email`
- `customer_phone`
- `gst_amount` (replaced with separate sgst + cgst)
- `total_amount` (replaced with grand_total)

#### Renamed Columns:
- `customer_name` → `client_name`
- `customer_address` → `client_address`

## Technical Details

### PDF Generation
- Uses `jsPDF` with `jspdf-autotable` for professional layouts
- A4 page size with 15mm margins
- Theme color from `company_profile.theme_color` (default: #185FA5)
- Logo handling with automatic base64 conversion and error handling
- Number-to-words conversion handles up to Crores

### Form Validation
- Customer name: Required
- At least one item: Required
- Item details (description, qty, unit price): Required
- All new fields: Optional but recommended

### GST Calculation
- SGST: 9% of subtotal (rounded)
- CGST: 9% of subtotal (rounded)
- Grand Total: Subtotal + SGST + CGST
- Calculation always rechecked on save

## Support & Troubleshooting

### Common Questions

**Q: Will my old quotations still work?**
A: After database migration, yes. The migration script updates old data to new format.

**Q: Can I still send quotations via WhatsApp?**
A: Yes! WhatsApp feature is unchanged.

**Q: Can I customize the PDF further?**
A: Yes, all PDF generation code is in `/app/(dashboard)/quotations/[id]/page.tsx` in the `handleDownloadPdf()` function.

**Q: What if I don't want to use the new fields?**
A: They're all optional except customer name. Leave them blank if not needed.

### If Something Goes Wrong

1. **Check browser console** for JavaScript errors
2. **Verify database migration** completed successfully
3. **Clear browser cache** and reload
4. **Check Supabase logs** for any errors
5. **Ensure company profile** has basic info set
