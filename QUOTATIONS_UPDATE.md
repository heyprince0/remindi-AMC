# Quotations Form & PDF Generation Update

## Summary of Changes

This update completely overhauls the quotation form and PDF generation to match professional Indian contractor quotation standards.

### Part 1: Form Updates

#### Changed Fields in Customer Section:
- **Removed**: Phone number and Email fields
- **Added**: District, State, Pin Code fields (all text inputs)

#### New Letter Details Section:
- **Subject** field: Single-line text input for quotation subject
  - Placeholder: "e.g. Regarding AC AMC works at above mention place"
  - Saved as `subject` column

- **Letter Body** field: Textarea (4 rows) for letter content
  - Placeholder: "e.g. Respected Sir/Madam, As per discussion held with you..."
  - Quick suggestion buttons to insert common lines:
    1. "Respected Sir/Madam,"
    2. "As per discussion held with you, regarding following works at above mention place. Details are given as below;"
  - Saved as `body_text` column

#### Bug Fixes:
- Fixed GST NaN bug by ensuring GST percentage is always a valid number
  - Changed calculation: `const gstRate_ = Number(gstRate ?? 18) / 100`

#### Files Modified:
- `/app/(dashboard)/quotations/new/page.tsx` - New quotation form
- `/app/(dashboard)/quotations/[id]/edit/page.tsx` - Edit quotation form
- `/lib/supabase.ts` - Updated Quotation type definition

## How to Apply Changes

### Step 1: Update Database Schema

⚠️ **IMPORTANT**: The old quotations table schema has changed. You must:

**Option A: Fresh Start (Recommended)**
1. Delete all old quotations data (if any)
2. Run the updated migration in Supabase:
   - Go to SQL Editor → New Query
   - Copy the updated `lib/db-migrations.sql` content
   - Click Run

**Option B: Migrate Existing Data**
If you have existing quotations, run this ALTER TABLE command:
```sql
ALTER TABLE quotations
ADD COLUMN IF NOT EXISTS quote_no TEXT,
ADD COLUMN IF NOT EXISTS client_name TEXT,
ADD COLUMN IF NOT EXISTS client_address TEXT,
ADD COLUMN IF NOT EXISTS client_city TEXT,
ADD COLUMN IF NOT EXISTS client_district TEXT,
ADD COLUMN IF NOT EXISTS client_state TEXT,
ADD COLUMN IF NOT EXISTS client_pin_code TEXT,
ADD COLUMN IF NOT EXISTS client_gstin TEXT,
ADD COLUMN IF NOT EXISTS subject TEXT,
ADD COLUMN IF NOT EXISTS body_text TEXT,
ADD COLUMN IF NOT EXISTS sgst NUMERIC(10, 2) DEFAULT 0,
ADD COLUMN IF NOT EXISTS cgst NUMERIC(10, 2) DEFAULT 0,
ADD COLUMN IF NOT EXISTS grand_total NUMERIC(10, 2) DEFAULT 0,
ADD COLUMN IF NOT EXISTS valid_till DATE,
DROP COLUMN IF EXISTS customer_email,
DROP COLUMN IF EXISTS customer_phone,
DROP COLUMN IF EXISTS customer_id,
DROP COLUMN IF EXISTS quotation_number,
DROP COLUMN IF EXISTS gst_amount,
DROP COLUMN IF EXISTS total_amount;
```

### Step 2: Update Table Data
If migrating existing data, map old columns to new ones:
```sql
UPDATE quotations
SET 
  client_name = customer_name,
  client_address = customer_address,
  quote_no = quotation_number,
  sgst = CASE WHEN include_gst THEN (subtotal * 0.09)::NUMERIC(10,2) ELSE 0 END,
  cgst = CASE WHEN include_gst THEN (subtotal * 0.09)::NUMERIC(10,2) ELSE 0 END,
  grand_total = CASE WHEN include_gst THEN subtotal + (subtotal * 0.18)::NUMERIC(10,2) ELSE subtotal END
WHERE client_name IS NULL;
```

### Step 3: No Code Deployment Changes
All code changes have been made. Just push to your repository:
```bash
git add .
git commit -m "feat: Rebuild quotation form and PDF generation for professional Indian contractor layout"
git push
```

The app will automatically use the new forms and PDF generation on next deploy.

## Important Notes

1. **Color Scheme**: PDF uses theme color from company profile. Default is `#185FA5` (blue)
2. **Logo Handling**: If logo URL fails to load, PDF generation continues without it (no error)
3. **Amount in Words**: Supports up to Crores in Indian Rupee format
4. **Quotation Numbers**: Generated as `QT-001`, `QT-002`, etc. based on creation order
5. **GST Calculation**: Always uses 9% SGST + 9% CGST (18% total) when enabled
